GPU-ASCII 开发者文档

GPU 加速的图像转 ASCII 工具 + 终端游戏引擎框架。
底层使用 Rust + wgpu compute shader 并行处理图像，通过 C ABI FFI 导出动态库，Python 端通过 ctypes 零依赖调用。

架构

脏区域检测机制

游戏模式下，每帧的 ASCII 网格与上一帧逐字符比较。只有发生变化的字符格才通过 cursor::MoveTo 定位重写，避免全屏刷新导致的闪烁。

// src/ascii/game_renderer.rs render_dirty() 核心逻辑 for y in 0..max_rows { for x in 0..max_cols { if new_grid[y][x] != prev_grid[y][x] { // 只光标定位到变化位置，写入新字符 output.execute(cursor::MoveTo(x, y))?; write!(output, "{}", new_ch)?; } } }

两个着色器

着色器文件输出用途
AsciiShadershaders/ascii.wgslVec<u32> 字符索引默认模式，ASCII 黑白输出
ColorShadershaders/ascii_color.wgslVec<ColorOutput> (索引 + RGB)--color 模式，彩色 ASCII 输出

ColorOutput 结构体: { char_idx: u32, r: u32, g: u32, b: u32 }，每个字符格同时输出字符索引和该区域的平均颜色。

环境要求

Rust 端

  • Rust toolchain (rustup)
  • Visual Studio Build Tools (MSVC linker)，Windows 必须
  • 支持 Vulkan / DX12 / Metal 的 GPU（wgpu 自动选择后端，优先 HighPerformance 适配器）

Python 端

  • Python >= 3.8
  • 无第三方依赖（纯 ctypes 调用动态库）
  • PIL / Pillow 仅平台跳跃游戏 (game.py) 需要

快速开始

静态图像转换

import sys
sys.path.insert(0, "python")   # 或者把 python/ 加入 PYTHONPATH

from gpu_ascii import GpuAscii, image_to_ascii

# 方式 1: 便捷函数（一行搞定）
text = image_to_ascii("photo.png", cell_size=8)
print(text)

# 方式 2: 自动适配终端大小
text = image_to_ascii("photo.png", fit_terminal=True)
print(text)

# 方式 3: 完整控制
gpu = GpuAscii()
result = gpu.convert(
    image_path="photo.png",
    cell_size=16,
    char_ramp=" .:;+=xX$#@█",
    use_color=False,
)
print(f"ASCII 尺寸: {result.width}x{result.height}")
print(result.text)

游戏模式（动态刷新）

from gpu_ascii import GpuAscii, GameRenderer, GameLoop

gpu = GpuAscii()
renderer = GameRenderer(gpu)
game_loop = GameLoop(target_fps=30)

renderer.init()  # 进入备用屏幕，隐藏光标
try:
    while game_loop.begin_frame():
        renderer.update_to_terminal("frame.png")
        game_loop.end_frame()
finally:
    renderer.cleanup()  # 恢复终端

GpuAscii — 核心转换器

GPU 上下文，封装 Rust 侧的 GpuAsciiHandle（包含 AsciiConverter）。负责图像到 ASCII 的转换。

class GpuAscii(lib_path: str | None = None)
方法签名说明
get_image_info(image_path: str) -> (int, int)返回图像 (width, height)，不进行 GPU 转换
calc_cell_size(img_w, img_h, term_w?, term_h?) -> int根据图像和终端尺寸计算最优 cell_size（考虑 2:1 字符高宽比）
convert(image_path, cell_size=8, char_ramp=" .:-=+*#%@", use_color=False) -> AsciiResultGPU 转换图像为 ASCII
convert_to_terminal(image_path, char_ramp, use_color, terminal_width?, terminal_height?, padding=1) -> AsciiResult自动适配终端大小转换

AsciiResult

@dataclass
class AsciiResult:
    text: str      # ASCII 文本（换行分隔）
    width: int     # 输出列数
    height: int    # 输出行数

char_ramp 规则

字符串从左到右对应从暗到亮的像素亮度。空格 = 最暗（背景），最后一个字符 = 最亮。字符数量影响亮度分级精度。GPU 着色器会对每个 cell_size×cell_size 区域计算 BT.709 感知亮度，然后映射到字符索引。

GameRenderer — 游戏模式渲染器

封装 Rust 侧的 GameRendererHandle（包含 AsciiConverter + GameRenderer + prev_frame）。支持单页面动态渲染，脏区域检测只重绘变化的字符格。支持 with 语句。

class GameRenderer(gpu: GpuAscii, char_ramp: str = " .:-=+*#%@")
方法签名说明
init()进入备用屏幕（EnterAlternateScreen），隐藏光标，清屏
init_debug()调试模式（不进入备用屏幕，隐藏光标，清屏，可与 input() 配合）
update(image_path, cell_size=8) -> FrameInfo加载图像 → GPU 转换 → 脏区域检测 → 渲染到终端
update_to_terminal(image_path, padding=1) -> FrameInfo自动计算 cell_size 后渲染
update_from_memory(rgba_data: bytes, w, h, cell_size, char_ramp) -> FrameInfo从内存 RGBA 数据渲染（适合程序化生成画面）
get_frame_count() -> int已渲染帧数
set_terminal_height(height: int)限制输出行数，防止滚动
cleanup()显示光标，重置颜色，退出备用屏幕（如果使用了 init()）

FrameInfo

@dataclass
class FrameInfo:
    dirty_cells: int   # 本帧变化的字符格数
    total_cells: int   # 总字符格数
    grid_width: int    # 网格列数
    grid_height: int   # 网格行数

update() 内部流程

update(image_path, cell_size) │ ├→ image::open(path) // 加载图像 ├→ converter.convert(img, cell) // GPU 转换 → Vec<Vec<char>> ├→ dirty_cells = 逐格比较 prev // 脏区域计算 ├→ renderer.render_dirty(grid) // 只重写变化的字符 └→ return FrameInfo
init() vs init_debug()：
init() 执行 EnterAlternateScreen + Clear(All) + Hide，适合正式游戏。
init_debug() 只执行 Clear(All) + Hide，不切换屏幕，适合开发调试。
两者不能混用，一个实例只能 init 一次（除非 cleanup 后重新 init）。
Rust 侧的 GameRenderer 实现了 Drop，析构时自动调用 cleanup()。

RawKeyboard — Win32 原始键盘

使用 GetAsyncKeyState 轮询硬件按键状态，支持方向键、空格等所有键。 Windows

  • 数字: '0' ~ '9'

  • 功能键: 'space', 'enter', 'esc', 'tab', 'backspace'

Scene / SceneManager — 场景栈

场景管理器，基于列表实现的栈结构，支持 push / pop / switch。

class Scene(ABC):
    name: str
    def on_enter(self): ...      # 进入场景时
    def on_exit(self): ...       # 离开场景时
    def on_pause(self): ...      # 被压入栈下方时
    def on_resume(self): ...     # 从栈下方恢复时
    def update(self, delta_time: float): ...  # 必须实现
    def render(self, renderer): ...           # 必须实现
    def set_data(key, value): ...             # 场景内数据存储
    def get_data(key, default=None): ...
class SceneManager()
方法说明

注意事项

1. 动态库必须先编译

Python 绑定依赖 Rust 编译产物 (gpu_ascii.dll / libgpu_ascii.so)。运行任何 Python 脚本前必须先：

cargo build --release

2. RawKeyboard 仅 Windows

rawkb.py 使用 Win32 API GetAsyncKeyState，非 Windows 平台所有方法返回 False。跨平台场景请用 InputManager（基于 msvcrt，功能较弱）。

3. GameRenderer 的 init() vs init_debug()

  • init() 执行 EnterAlternateScreen + Clear(All) + cursor::Hide，cleanup() 时执行 LeaveAlternateScreen
  • init_debug() 只执行 Clear(All) + cursor::Hide，不切换屏幕
    • 两者不能混用，一个实例只能 init 一次（除非 cleanup 后重新 init）。

4. 资源释放

  • GameRenderer 支持 with 语句，推荐使用
  • GpuAscii 在 __del__ 中调用 gpu_ascii_free(handle)
  • GameRenderer 在 __del__ 中调用 game_renderer_free(handle)
  • Rust 侧 GameRenderer 实现了 Drop trait，析构时自动 cleanup()
  • 不要在 GameRenderer 的 __del__ 之前调用 GpuAscii 的 __del__，否则会 UAF

5. cell_size 含义

cell_size 表示每个 ASCII 字符对应原图的像素块大小。着色器对该区域内的所有像素计算 BT.709 加权平均亮度。垂直方向使用 cell_size × 2（ASCII 字符高宽比约 2:1）。

  • 值越小 → 输出越大越精细（更多 GPU 线程）
  • 值越大 → 输出越小越粗糙
  • calc_cell_size() 自动计算最优值

6. 线程安全

所有 API 均非线程安全。wgpu 上下文绑定到创建线程。如果需要多线程，每个线程应创建独立的 GpuAscii 实例。

7. GPU 适配器选择

Rust 侧 GpuContext::new() 使用 PowerPreference::HighPerformance 请求适配器，优先选择独立 GPU。如果只有集显，会自动回退。多 GPU 环境下可通过 WGPU_ADAPTER_NAME 环境变量指定。

8. game.py 额外依赖

平台跳跃游戏 (python/games/platformer/game.py) 需要：

  • Pillow（PIL）— 画面合成（背景 / 平台 / 金币 / 玩家 → RGBA 画布）
  • rawkb.py — 键盘输入（仅 Windows）
pip install Pillow

9. GIF 帧文件命名

SpriteSheet.from_gif_frames() 和部分示例期望 GIF 帧文件名格式：

frame_00_delay-0.05s.gif
frame_01_delay-0.05s.gif
...

正则: frame_(\d+)_delay-([\d.]+)s\.gif
